## 一、视觉语言系统

### 1.1 色彩体系

| Token | 值 | 用途 |
|-------|-----|------|
| `--bg-primary` | `#F5EFE4` | 页面主背景（暖白米色） |
| `--bg-card` | `rgba(255,255,255,0.9)` | 卡片背景（半透明白） |
| `--text-primary` | `#0C0A09` | 主标题文字（暖黑） |
| `--text-body` | `#181411` | 正文文字 |
| `--text-hint` | `#8B8580` | 辅助文字 |
| `--border-subtle` | `rgba(231,229,228,0.8)` | 卡片/分割边框 |
| `--shadow-warm` | `rgba(57,45,26,0.14)` | 暖棕色阴影 |
| `--accent` | `#0C0A09` | 主按钮/强调色（近黑） |
| `--accent-secondary` | `#444038` | 次要文字/标签 |

**原则:** 暖白 > 暖灰 > 近黑，克制用色，不刺眼。无冷色科技感。

### 1.2 圆角体系

| Token | 值 | 用途 |
|-------|-----|------|
| `--radius-lg` | `30px` | 主卡片/主容器 |
| `--radius-md` | `16px` | 子卡片/SKU 卡片 |
| `--radius-full` | `9999px` | 按钮 pill / 标签 / 头像 |

### 1.3 字体层级

| 层级 | 字号 | 字重 | 行高 | 用途 |
|------|------|------|------|------|
| H1 | 62px | 900 | 65px | 页面大标题 |
| H2 | 30px | 900 | 36px | 区域标题 |
| H3 | 20px | 700 | - | 模块标题 |
| Body | 16px | 400 | - | 正文 |
| Caption | 14px | - | - | 辅助/标签 |
| Tiny | 12px | - | - | 提示小字 |

**字体栈:** `-apple-system, BlinkMacSystemFont, "Segoe UI", "PingFang SC", "Hiragino Sans GB", "Microsoft YaHei", Arial, sans-serif`

### 1.4 阴影体系

| Token | 值 | 用途 |
|-------|-----|------|
| `--shadow-card` | `0 28px 90px rgba(57,45,26,0.14)` | 主卡片阴影 |
| `--shadow-float` | `0 14px 36px rgba(57,45,26,0.10)` | 悬浮层阴影 |
| `--shadow-btn` | `0 1px 2px rgba(0,0,0,0.05)` | 按钮微阴影 |

**关键:** 所有阴影用暖棕色系 `rgba(57,45,26,...)`，不用灰色阴影。

### 1.5 间距体系

| Token | 值 |
|-------|-----|
| Page max-width | 1680px |
| Page horizontal pad | 32px |
| Nav height | 88px |
| Card padding | 32px |
| Section gap | 48px |

## 三、电商模块：统一工作台（精修工坊折叠版）

### 3.1 简单模式 vs 专业模式

| 维度 | 简单模式（默认） | 专业模式（展开） |
|------|-----------------|-----------------|
| 入口显示 | 一句话输入框 | 一句话输入框 + 5步参数面板 |
| 参数控制 | 走默认预设（白底×1 / 主图1:1×5 / 3:4×5 / 透明×1 / SKU-无） | 所有参数用户自定义 |
| 参考图 | 选传（最多4张） | 强制双栏：左边实拍 / 右边风格 |
| 目标人群 | 卖家老板 | 美工设计师 |

## 七、数据流（不变）

```
用户输入一句话
  → EcMode 使用预设规则填充默认参数
  → 如果展开专业面板，用户参数覆盖默认
  → POST /api/ecommerce-preview (参数同现在)
  → 返回 outline + imageTypes
  → POST /api/generate-ecommerce (SSE)
  → 流式出图 + 显示
```

**用户身份流:** 当前前端 fetch 不带 email，后端 requireCredits 已写但未挂载。本次 UI 重构不涉及。
