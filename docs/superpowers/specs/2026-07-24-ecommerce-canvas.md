# 电商无限画布重构规格

## 目标

把当前“图片节点展示器”升级为面向电商素材生产的无限画布：用户可以按商品用途管理生成结果，批量编辑详情图，建立可理解的素材关系，并把设计方案二次编辑后重新生成。

## 用户问题与产品逻辑

电商用户关心的是一套可交付的转化素材，而不是孤立的图片。每一张图都要回答“它用于什么位置、解决什么购买疑虑、如何和其他素材组合、能否再次修改和导出”。因此画布的核心对象是带业务语义的资产，不是普通 DOM 图片。

参考案例共同体现了五段生产链：

1. 输入商品图和参考素材。
2. AI 分析商品和平台目标，形成可编辑的设计方向。
3. 按主图、详情图、SKU 等用途生成有编号的结果。
4. 在画布中对单张或多张结果进行批处理和二次修改。
5. 导出为平台可用的图片包或长详情图。

## 交互规格

### 画布

- 画布只保留一套缩放控件：缩小、当前比例、放大、适配视口；不再显示重复的缩放入口。
- 删除左侧悬浮工具栏，画布从页面左边缘开始布局。
- 空白区域拖动平移，滚轮以指针为中心缩放。
- 空白区域按下并拖动生成框选矩形；框内节点全部加入多选。
- Ctrl/Cmd 点击切换节点选择，Shift 点击在当前选择基础上追加。
- 多选节点可以一起拖动、下载、删除、改分类、改名。
- 节点拖动只改变位置，不能改变已有关系，也不能把图片与“主图”等文字标签错误绑定。
- 每个节点左右显示连接端口。只允许从端口拖出连线，连线落到另一个端口后建立关系；点击空白或 Escape 取消。
- 连接关系持久化在画布状态中，包含 `type`：`reference`（引用素材）、`variant`（生成变体）、`merge`（合并产物）。
- 删除节点时同步删除相关边。
- 快捷键：Delete/Backspace 删除、Ctrl/Cmd+A 全选、Escape 清空选择或取消连线、F 适配视口。

### 电商资产

每个节点至少包含：

```js
{
  id: string,
  assetId: string,
  url: string,
  name: string,
  group: '主图' | '详情图' | 'SKU' | '素材',
  role: string,
  ratio: string,
  usage: string,
  sourceKey: string,
  sourceDirectionId?: string,
  editable: boolean,
  x: number,
  y: number,
  w: number,
  h: number
}
```

默认命名规则：

- `white_bg` → `白底首图`
- `main_text` → `场景主图`
- `main_3x4` → `竖版主图`
- `sku` → `SKU规格图`
- `detail_slice_size` → `尺寸标注图`
- `detail_slice_scene` → `场景使用图`
- `detail_slice_qc` → `品质背书图`
- `detail_slice_compare` → `优势对比图`
- `detail_slice_feature` → `细节功能图`
- `detail_slice_care` → `使用维护图`

同类副本追加 `-01`、`-02` 编号。用户可以直接编辑名称和分组。

### 工具栏与批处理

- 单选节点显示：下载、重命名、改分类、去背、反推提示词、二次生成、复制链接、删除。
- 多选节点显示：批量下载、批量改分类、批量删除；当全部为详情图且数量不少于 2 时显示“合并为详情长图”。
- 合并产物加入画布，命名为 `详情长图-01`，分组为 `详情图`，并用 `merge` 边连接来源节点。
- 未接入真实服务的高级能力不显示为可点击的假按钮；现有真实 API（去背、反推提示词、重生成、长图合成）继续复用。

### 设计方案二次编辑

方案方向包含 `id`、标题、用途、构图说明、文案要求、比例、目标平台、参考图和生成参数。方案卡可以打开编辑表单；保存后保留原方向 ID，重新生成的资产通过 `sourceDirectionId` 关联回方案。

## 视觉规格

- 维持现有浅色点阵背景和紫色主色，但减少漂浮装饰。
- 顶部工具栏左侧为返回、商品名、资产统计和分组筛选；右侧为单一缩放组、新建生图和批处理动作。
- 节点卡片顶部显示分组徽标和编号，底部显示名称、比例、用途；选中时显示紫色描边、复选框和左右连接端口。
- 选中多张图时在顶部显示明确的“已选 N 张”和批处理动作。
- 底部保留“引用素材 + 提示词 + 重新生成”能力，并显示当前引用的节点缩略图。

## 验收标准

1. `npm run test` 通过，新增画布状态测试覆盖命名、框选、批量移动、端口连线和详情合并边。
2. `npm run build` 通过。
3. 代码中不再渲染左侧固定悬浮工具栏，也不再存在两套缩放控件。
4. 画布中拖动节点不会改变 `connections`，端口连线可以建立和删除关系。
5. Ctrl/Cmd 多选与空白框选都能选中多个节点；多选节点能一起移动。
6. 详情图多选后出现合并按钮，合并成功后新节点显示 `详情长图-01` 等业务名称，并且来源边为 `merge`。
7. 主图、详情图、SKU、素材在节点卡片和筛选器中可区分。
8. 设计方向可再次编辑，保存后重新生成的节点带有 `sourceDirectionId`。
9. 部署后 `https://shuimg.cn` 返回 200，后端 `/health` 返回 `ok: true`，PM2 进程在线。
